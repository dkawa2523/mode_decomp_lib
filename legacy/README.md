# legacy/（任意）

このプロジェクトは「0から作る」前提ですが、既存リポジトリ（旧実装）の **アルゴリズム部分だけ** を参照して移植するとP0が速く安定します。

## 推奨
- 旧リポジトリをこの `legacy/` 配下にコピー（または git submodule / subtree）してください。
- Codex Autopilot は **workspace-write** で動くため、基本的にこのプロジェクトフォルダ内のファイルしか参照できません。
  そのため、旧コードを参照したい場合は `legacy/` 配下に置くのが確実です。

## 移植対象例
- Zernike展開（基底生成、係数推定、逆変換）
- データ入出力（フォーマット変換、mask処理）
- 可視化（係数スペクトル、再構成比較）

**注意**: 旧コードを丸ごとコピーして “二重実装” にするのは禁止です。
必要最小限を `src/` の契約に合わせて移植してください（docs/14,15,16参照）。
