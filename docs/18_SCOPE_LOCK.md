# スコープロック（タスクが増え続けないための運用）

既存リポジトリの大規模改修では、タスクが連鎖して増え続け、Autopilotが終わらない問題が発生しやすいです。
本プロジェクトはこれを避けるため、以下を **不変運用** とします。

## ルール
1. **P0は完了条件が明確な“最小完成形”** まで。機能追加はしない。
2. Autopilot実行中に `work/queue.json` に **新規タスクIDを追加しない**（task creep禁止）。
   - 足りない作業が見つかった場合は、当該タスク内でTODOにまとめ、必要なら **別RFC** を作る。
3. 拡張（P1/P2）は **別queue** で管理する：
   - `work/queue_p1.json`
   - `work/queue_p2.json`
4. 例外的にP0中に追加が必要な場合：
   - ユーザーが明示的に許可した場合のみ
   - 追加は `work/queue_patch.json` に行い、Autopilotを止めて合意する

## 目的
- 途中で全体をやり直す大規模リファクタを避ける
- 比較基盤（I/O、artifact、評価）がブレないよう固定する
